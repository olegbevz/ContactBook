Для реализации тестового задания была выбрана платформа ASP.NET MVC из соображений наибольшего соответствия технологиям компании.

Для хранения данных используется xml-файл в целях экономии времени на выполнение тестового задания.
При дальнейшем развитии подобного проекта, данные следует хнанить в реляционной БД (например MS SQL), 
доступ к данным осуществлять с помощью ORM (NHibernate, Entity Framework).

В процессе разработки программы были реализованы следующие паттерны проектирования: 
1. Модель - Представление - Контроллер (ASP.NET)
2. Репозитория (Наследники IRepository)
3. Синглтон (MemoryRepository.Instance)
4. Абстрактная фабрика (IoC : Ninject)
